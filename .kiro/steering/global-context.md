For cross-project context, ecosystem understanding, shared conventions, and architecture decisions, consult the **kaelint-knowledge-base** repository at `/Users/tabkamac/private/dev/git/kaelint-knowledge-base/`.

## When to Reference the Knowledge Base

- Understanding how this project connects to the website-business and CRM
- Checking shared coding standards, git conventions, or testing patterns
- Understanding the pricing model, brand identity, or operator workflows
- Security practices and email sending best practices
- CI/CD pipeline definitions
